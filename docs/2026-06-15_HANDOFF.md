## スマホ側で分かったこと

### MacroDroid

MacroDroidは実機で「無料 残り7日間」と表示された。

無料運用を優先するため、MacroDroidは採用しない。

### Open Camera

Open Cameraを使い、レシート画像を専用フォルダに保存する方針に変更した。

保存先:

```text
DCIM/receipt-ocr
```

FolderSyncから見るパス:

```text
/sdcard/DCIM/receipt-ocr/
```

または:

```text
/storage/emulated/0/DCIM/receipt-ocr/
```

### FolderSync

FolderSyncは最初、以下のエラーで失敗した。

```text
ファイルにアクセスする権限がありません
```

原因は、FolderSyncがAndroidローカルの画像ファイルへアクセスできていなかったこと。

対応:

```text
Android設定
  -> アプリ
  -> 特別なアプリアクセス
  -> すべてのファイルへのアクセス
  -> FolderSync を許可
```

この設定後、FolderSyncの同期カードが緑色になり、同期成功を確認した。

### 2. Mac上の実パスを確認

Finderで `家計簿/receipt-ocr` を開き、ターミナルで使える実パスを確認する。

候補例:

```text
~/Google Drive/家計簿/receipt-ocr
~/Library/CloudStorage/GoogleDrive-.../My Drive/家計簿/receipt-ocr
```

Google Drive for desktopのバージョンや設定によってパスが違う可能性がある。

### 3. `config/config.json` を更新

実パスに合わせて `drive.source_dir` を更新する。

例:

```json
"drive": {
  "enabled": true,
  "source_dir": "~/Google Drive/家計簿/receipt-ocr",
  "after_import": "archive",
  "archive_dir": "~/Google Drive/家計簿/receipt-processed"
}
```

`archive_dir` は必要ならGoogle Drive側に作る。

例:

```text
家計簿/receipt-processed
```

## 注意点

Google Drive for desktopはストリーミング前提。

`receipt-ocr` の `source_dir` はGoogle Drive for desktopの同期設定ではない。Mac上に見えているGoogle Driveフォルダのうち、receipt-ocrが読むフォルダを指定するだけ。

Drive側のアップロード元画像をどう扱うかは `after_import` で決まる。

```json
"after_import": "archive"
```

なら、取り込み後にDrive側の画像を `archive_dir` へ移動する。

```json
"after_import": "delete"
```

なら、取り込み後にDrive側の画像を削除する。

最初は `archive` 推奨。
