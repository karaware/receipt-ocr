## 今回の修正: 税・値引き・手数料を主カテゴリへ含める

### 背景

従来は消費税、値引き、手数料、端数を `調整` 大カテゴリへ入れていた。この場合、たとえば食費のレシートで100円引きがあると、食費の集計が実際の支払額より100円多くなる。

### 新しい分類ルール

1. レシートの通常商品明細を大カテゴリごとに合計する。
2. 合計金額が最も大きい大カテゴリを、そのレシートの主カテゴリとする。
3. 消費税・値引き・手数料・端数は、主カテゴリの `値引き・税・手数料` 小カテゴリへ入れる。
4. 通常商品がなく主カテゴリを決められない例外だけ `調整` を使う。

例:

```text
かけ(大)             630円  食費 / 外食
かしわ天              220円  食費 / 外食
鮭おむすび            170円  食費 / 外食
5枚天ぷら100円引     -100円  食費 / 値引き・税・手数料
合計                  920円
```

この場合、食費の大カテゴリ集計も実際の支払額と同じ920円になる。

### 実装内容

- OCR/LLMへの指示を更新し、主カテゴリへ分類するよう変更した。
- LLM結果の検証で、税・値引き・手数料・端数が主カテゴリ以外なら拒否して再試行する。
- Firestoreの既存・新規の全支出カテゴリへ `値引き・税・手数料` 小カテゴリを自動追加する。
- Webのカテゴリ追加・編集時にも、支出カテゴリにはこの小カテゴリを自動的に維持する。
- Webの確認画面で差額明細を追加する場合も、主カテゴリを初期選択する。
- 税・値引き・手数料の明細から商品名分類ルールを学習しないようにした。

既存の確定済み取引は変更しない。反映後にOCRする新規レシートから適用する。
